# Journal Entry 006 - 2026-03-04 - Add --list flag to rem update

## Goal

Pick and implement the first open GitHub issue: add a `--list` flag to `rem update` so users can move reminders between lists.

## What Changed

- `cmd/rem/commands/update.go`: Added `updateList` var, `--list/-l` flag registration, and `cmd.Flags().Changed("list")` check in `RunE`
- `internal/service/reminders.go`: Added `"list"` case in `UpdateReminder` switch that sets `input.ListName`, added `input.ListName != nil` to `hasEventKitUpdates` guard
- `website/content/docs/commands.md`: Added `--list` row to `rem update` flag table
- `skills/rem-cli/references/commands.md`: Added `--list` row to flag table and example usage in code block
- Released as v0.8.0

## Key Insights

- **go-eventkit `UpdateReminderInput` already has `ListName *string`** — moves the reminder natively, no delete+recreate needed. The issue notes said "delete + recreate under the hood" but that's not how go-eventkit works.
- **Interactive mode already handled list moves** — `runUpdateInteractive` already had list selection wired up (`updates["list"] = listName`), but the service `UpdateReminder` had no `"list"` case so it was silently ignored. Non-interactive path was the only gap.
- **Release order gotcha (re-confirmed)**: Accidentally tagged before pushing docs commit during this session. Deleted tag with `git push origin --delete v0.8.0 && git tag -d v0.8.0`, pushed docs, then re-tagged. Always: push commits → tag → push tag → build.

## Decisions Made

- No explicit list validation in the command layer — let go-eventkit return the error if the list doesn't exist. Keeps the code minimal and consistent with other commands.
- Used `-l` short flag (same as `rem add --list`) for consistency across the CLI surface.
