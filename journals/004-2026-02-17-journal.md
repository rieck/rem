# Journal Entry 004 - 2026-02-17 - Agent Skills & Background Update Check

## Goal

Port the agent skills distribution system and background update check from the `ical` project to `rem`. This gives users `rem skills install/uninstall/status` commands and automatic new-version notifications.

## What Changed

### New packages
- **`internal/skills/`** — Agent skill install/uninstall/status/detect logic. Ported from ical with `rem-cli` naming. Handles symlink replacement (dev setups), version tracking via `.rem-version` file, multi-agent support (Claude Code + Codex CLI).
- **`internal/update/`** — Background update check against GitHub releases API. 24h file cache at `~/.cache/rem/update-check`, 2s HTTP timeout, fire-and-forget goroutine pattern. Semver comparison with pre-release suffix stripping.
- **`skills.go`** (root) — `go:embed skills/rem-cli` directive exposing `EmbeddedSkills` as `embed.FS`.

### New CLI commands
- **`cmd/rem/commands/skills.go`** — `rem skills install/uninstall/status` with `--agent claude|codex|all` flag. Interactive `huh.NewMultiSelect` picker when no flag provided. Non-TTY auto-detects installed agents.

### Modified files
- **`cmd/rem/commands/root.go`** — Added `PersistentPreRun` (starts background update goroutine + NO_COLOR handling) and `PersistentPostRun` (prints update notice + skills staleness notice to stderr). `shouldCheckForUpdate()` skips for: `REM_NO_UPDATE_CHECK=1`, dev builds, json output, non-TTY, meta commands (version/completion/skills).
- **`scripts/install.sh`** + **`website/static/install`** — Post-install prompt for skill installation using `read -r answer < /dev/tty` for curl-pipe-bash compatibility.
- **Skill files** — Added skills commands to `SKILL.md` command table and `references/commands.md`. Added `REM_NO_UPDATE_CHECK` env var docs.
- **Deleted `skills/rem-cli/references/go-api.md`** — Stale since `pkg/client/` removal.
- **Website** — Added skills sections to `getting-started.md` and `commands.md`.
- **`README.md`** — Added AI Agent Skills section and updated architecture tree.
- **`CLAUDE.md`** — Added skills architecture docs, `charmbracelet/huh` library.

### New dependency
- `charmbracelet/huh` v0.8.0 — Interactive multi-select for agent picker.

## Key Insights

- **Direct port worked cleanly** — The ical skills/update packages were well-abstracted. Only needed to change string constants (`ical-cli` → `rem-cli`, `BRO3886/ical` → `BRO3886/rem`, `~/.cache/ical/` → `~/.cache/rem/`).
- **`versionStr` vs `Version`** — ical uses a `SetVersionInfo()` pattern with private `versionStr`. rem uses exported `Version` set directly from `main.go`. Skills and root commands just reference `Version` directly since they're in the same package. No need to add a `versionStr` alias.
- **`isInteractive()` name collision** — rem already had potential for naming conflicts in the commands package, so the skills helper was named `isInteractiveSkills()`.
- **`PersistentPreRun` addition** — rem's root command didn't previously have `PersistentPreRun`/`PersistentPostRun`. Added both. The `NO_COLOR` handling was moved into `PersistentPreRun` (previously only checked at flag level).
- **Update check is non-blocking** — Uses buffered channel + `select/default` in PostRun. If the goroutine hasn't finished by command completion, the notice is silently skipped. No user-visible latency.

## Decisions Made

- **Same agent targets as ical** — Claude Code (`~/.claude/skills/`) and Codex CLI (`~/.agents/skills/`). Matches the Agent Skills standard.
- **`REM_NO_UPDATE_CHECK`** env var (not `NO_UPDATE_CHECK`) to avoid conflicts with other tools.
- **Skills staleness notice** — If installed skill version doesn't match binary version, prints a yellow notice to stderr. Only shown once even if multiple agents have stale skills.
- **Deleted go-api.md** rather than updating it — the file pointed to the removed `pkg/client/` package. SKILL.md already links to go-eventkit directly.
