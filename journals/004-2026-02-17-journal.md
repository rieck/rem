# Journal Entry 004 - 2026-02-17 - Agent Skills & Background Update Check

## Goal

Port the agent skills distribution system and background update check from the `ical` project to `rem`. This gives users `rem skills install/uninstall/status` commands and automatic new-version notifications.

## What Changed

### New packages
- **`internal/skills/`** ‚Äî Agent skill install/uninstall/status/detect logic. Ported from ical with `rem-cli` naming. Handles symlink replacement (dev setups), version tracking via `.rem-version` file, multi-agent support (Claude Code + Codex CLI).
- **`internal/update/`** ‚Äî Background update check against GitHub releases API. 24h file cache at `~/.cache/rem/update-check`, 2s HTTP timeout, fire-and-forget goroutine pattern. Semver comparison with pre-release suffix stripping.
- **`skills.go`** (root) ‚Äî `go:embed skills/rem-cli` directive exposing `EmbeddedSkills` as `embed.FS`.

### New CLI commands
- **`cmd/rem/commands/skills.go`** ‚Äî `rem skills install/uninstall/status` with `--agent claude|codex|all` flag. Interactive `huh.NewMultiSelect` picker when no flag provided. Non-TTY auto-detects installed agents.

### Modified files
- **`cmd/rem/commands/root.go`** ‚Äî Added `PersistentPreRun` (starts background update goroutine + NO_COLOR handling) and `PersistentPostRun` (prints update notice + skills staleness notice to stderr). `shouldCheckForUpdate()` skips for: `REM_NO_UPDATE_CHECK=1`, dev builds, json output, non-TTY, meta commands (version/completion/skills).
- **`scripts/install.sh`** + **`website/static/install`** ‚Äî Post-install prompt for skill installation using `read -r answer < /dev/tty` for curl-pipe-bash compatibility.
- **Skill files** ‚Äî Added skills commands to `SKILL.md` command table and `references/commands.md`. Added `REM_NO_UPDATE_CHECK` env var docs.
- **Deleted `skills/rem-cli/references/go-api.md`** ‚Äî Stale since `pkg/client/` removal.
- **Website** ‚Äî Added skills sections to `getting-started.md` and `commands.md`.
- **`README.md`** ‚Äî Added AI Agent Skills section and updated architecture tree.
- **`CLAUDE.md`** ‚Äî Added skills architecture docs, `charmbracelet/huh` library.

### New dependency
- `charmbracelet/huh` v0.8.0 ‚Äî Interactive multi-select for agent picker.

## Key Insights

- **Direct port worked cleanly** ‚Äî The ical skills/update packages were well-abstracted. Only needed to change string constants (`ical-cli` ‚Üí `rem-cli`, `BRO3886/ical` ‚Üí `BRO3886/rem`, `~/.cache/ical/` ‚Üí `~/.cache/rem/`).
- **`versionStr` vs `Version`** ‚Äî ical uses a `SetVersionInfo()` pattern with private `versionStr`. rem uses exported `Version` set directly from `main.go`. Skills and root commands just reference `Version` directly since they're in the same package. No need to add a `versionStr` alias.
- **`isInteractive()` name collision** ‚Äî rem already had potential for naming conflicts in the commands package, so the skills helper was named `isInteractiveSkills()`.
- **`PersistentPreRun` addition** ‚Äî rem's root command didn't previously have `PersistentPreRun`/`PersistentPostRun`. Added both. The `NO_COLOR` handling was moved into `PersistentPreRun` (previously only checked at flag level).
- **Update check is non-blocking** ‚Äî Uses buffered channel + `select/default` in PostRun. If the goroutine hasn't finished by command completion, the notice is silently skipped. No user-visible latency.

## Decisions Made

- **Same agent targets as ical** ‚Äî Claude Code (`~/.claude/skills/`) and Codex CLI (`~/.agents/skills/`). Matches the Agent Skills standard.
- **`REM_NO_UPDATE_CHECK`** env var (not `NO_UPDATE_CHECK`) to avoid conflicts with other tools.
- **Skills staleness notice** ‚Äî If installed skill version doesn't match binary version, prints a yellow notice to stderr. Only shown once even if multiple agents have stale skills.
- **Deleted go-api.md** rather than updating it ‚Äî the file pointed to the removed `pkg/client/` package. SKILL.md already links to go-eventkit directly.

## Session 2 ‚Äî huh-based Interactive Mode for All Commands

## Goal

Replace all raw `bufio.NewReader` interactive prompts with polished `charmbracelet/huh` forms. Add `-i/--interactive` flag with multi-select batch operations to every command that benefits from it.

## What Changed

### New file
- **`cmd/rem/commands/huh_helpers.go`** ‚Äî Shared helpers used across all interactive commands:
  - `huhTheme()` ‚Üí `huh.ThemeCatppuccin()` (single source of truth, matches skills.go)
  - `isTTY()` ‚Üí stdin terminal check (replaces `isInteractiveSkills()`)
  - `requireInteractive()` ‚Üí returns error if not TTY
  - `reminderLabel(r)` ‚Üí formats `"Title [List] (due: date) ‚öë"` for selects, truncates at 50 chars
  - `reminderMultiSelect(title, reminders)` ‚Üí huh MultiSelect returning selected IDs
  - `reminderSelect(title, reminders)` ‚Üí huh Select for single reminder pick
  - `listSelect(title)` ‚Üí fetches lists + huh Select for list name
  - `huhConfirm(title)` ‚Üí huh Confirm returning `(bool, error)`

### Modified files (8 files, +1076/-190 lines)
- **`interactive.go`** ‚Äî Complete rewrite. Main menu is now `huh.Select` loop with options: create, list, complete, delete, flag/unflag, manage lists, quit. Add form uses huh inputs/selects/confirm. Added sub-menus `runFlagMenuInteractive()` and `runListMgmtMenuInteractive()` that dispatch to per-command helpers. Removed all `bufio` usage.
- **`complete.go`** ‚Äî Added `--interactive/-i`, `--list/-l`, `--flagged` flags. Custom Args validator allows 0 args when `-i`. `runCompleteInteractive(uncomplete bool)` fetches filtered reminders ‚Üí multi-select ‚Üí batch complete/uncomplete. Same helper shared by `uncompleteCmd`.
- **`delete.go`** ‚Äî Added `--interactive/-i`, `--list/-l`, `--flagged` flags. Single-delete now uses `huhConfirm` when TTY (falls back to requiring `--force` for non-TTY). `runDeleteInteractive()` uses multi-select + confirm before batch delete.
- **`flag.go`** ‚Äî Added `--interactive/-i`, `--list/-l` to both `flagCmd` and `unflagCmd`. `runFlagInteractive(listName)` fetches incomplete reminders ‚Üí multi-select ‚Üí batch flag. `runUnflagInteractive(listName)` fetches flagged reminders ‚Üí multi-select ‚Üí batch unflag.
- **`update.go`** ‚Äî Implemented the already-declared `--interactive` flag. Args changed to `RangeArgs(0,1)` when `-i`. `runUpdateInteractive(idArg)`: if no ID, uses `reminderSelect()` picker; fetches current reminder; pre-populates all huh form fields; diffs against original to build `updates` map; only sends changed fields.
- **`lists.go`** ‚Äî Added `-i` to `create`, `rename`, `delete` subcommands. `runListCreateInteractive()` uses huh Input. `runListRenameInteractive()` uses `listSelect()` + huh Input. `runListDeleteInteractive()` uses `listSelect()` + `huhConfirm()`. Single-delete also switched from bufio to `huhConfirm`.
- **`skills.go`** ‚Äî Replaced `isInteractiveSkills()` call with `isTTY()`. Removed the now-dead `isInteractiveSkills()` function.

## Key Insights

- **huh form pre-population** ‚Äî huh uses Go variable binding (`Value(&myVar)`), so setting the variable before `form.Run()` pre-populates the field. This made the update form straightforward ‚Äî just initialize all bound vars from the fetched reminder.
- **Args validation pattern** ‚Äî Using inline `func(cmd, args)` closures that check the interactive flag works well: `if interactive { return MaximumNArgs(0) }; return ExactArgs(1)`. This preserves strict validation for CLI usage while allowing no args in interactive mode.
- **Confirmation replacement** ‚Äî For non-interactive single deletes, the old bufio `(y/N)` prompt was replaced with `huhConfirm` when TTY. For non-TTY (piped stdin), we now require `--force` instead of silently reading from pipe. This is safer.
- **ErrUserAborted handling** ‚Äî Every `form.Run()` call checks for `huh.ErrUserAborted` (Ctrl+C) and prints "Cancelled." + returns nil. Consistent UX across all commands.
- **Interactive helpers accept filter params** ‚Äî `runFlagInteractive(listName string)` and `runDeleteInteractive()` read from package-level flag vars. The interactive menu dispatchers pass empty strings. This allows both `rem flag -i --list Work` and the interactive menu to share the same code path.

## Decisions Made

- **Theme: `ThemeCatppuccin()`** ‚Äî Already used in skills.go, keeps visual consistency.
- **Multi-select label format** ‚Äî `"Title [List] (due: Jan 02, 15:04) ‚öë"` with 50-char title truncation. Enough context to identify reminders without horizontal overflow.
- **Delete non-TTY behavior** ‚Äî Changed from silently reading stdin (could accidentally delete) to requiring `--force`. Safer for scripts.
- **Update diff-based** ‚Äî Only sends changed fields to `UpdateReminder()`, avoiding unnecessary AppleScript calls for flagged when it hasn't changed.
- **All interactive helpers are standalone functions** ‚Äî Reusable from both `rem <cmd> -i` and the `rem interactive` menu. The menu is just a thin dispatcher.
