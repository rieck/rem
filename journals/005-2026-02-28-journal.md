# Journal Entry 005 - 2026-02-28 - OpenClaw Agent Support

## Goal

Add OpenClaw as a supported agent target for `rem skills install/uninstall`, alongside the existing `claude` and `codex` targets.

## What Changed

- **`internal/skills/skills.go`** — Added `OpenClaw` to `DefaultTargets()`: key `openclaw`, base dir `~/.openclaw/skills`
- **`internal/skills/skills_test.go`** — Updated `TestDefaultTargets` to expect 3 targets; added assertions for openclaw key, name, and base dir (uses `strings.Contains` for portability)
- **`cmd/rem/commands/skills.go`** — Updated help text, `--agent` flag descriptions, and error message in `resolveAgentFlag` to include `openclaw`
- **`skills/rem-cli/SKILL.md`** — Updated install section: added curl install method, added `rem skills install --agent openclaw` example
- **`skills/rem-cli/references/commands.md`** — Added `openclaw` to install/uninstall examples, flag descriptions, and supported targets table with path `~/.openclaw/skills/rem-cli/`
- **`website/content/docs/commands.md`** — Added openclaw to install example and updated `--agent` flag description
- **`website/content/docs/getting-started.md`** — Added OpenClaw to agent list mention and added `--agent openclaw` example

## Key Insights

- The `DefaultTargets` pattern is clean — adding a new agent is a single struct literal in one function. No switch cases, no registration, just append to the slice.
- Test used `strings.Contains` for the base dir assertion rather than hardcoding the full path — better since home dir varies per machine.
- All tests passed immediately, no surprises.

## Decisions Made

- **`~/.openclaw/skills/`** as the install path — mirrors the convention of `~/.claude/skills/` and `~/.agents/skills/`. Assumed standard based on OpenClaw naming.
- No changes to `resolveAgentFlag` logic beyond adding `openclaw` to the error message — the existing loop already handles any key dynamically via `DefaultTargets`.
